1. Escenario elegido:
La opción elegida por el equipo es la C, debido a que nos resultó interesante y un escenario a el cual no nos sometimos antes, nos resulta un desafío que nos puede hacer aprender mucho más, debido a que con anterioridad ya hemos realizado turneros o sistemas de gestion de inventario.

2. Metodología elegida: 
La metodología de desarrollo elegida es scrum, debido a que consideramos que para este proyecto se ajusta mucho mas a nuestros horarios, el hecho de realizar reuniones cada determinado tiempo nos va a ayudar a ajustar nuestra forma de trabajo y a ser mas eficientes con las tareas que le corresponden a cada uno.

4. Acuerdos de trabajo: 
Nuestros principales acuerdos como equipo de trabajo son: comunicación via Whatsapp, Reuniones cada 2 días realizadas en Discord, la frecuencia de comitts será cada vez que comprobemos que el módulo a agregar cumple con las funciones requeridas. El criterio para mover las tarjetas será estrictamente cuando la tarea esté en curso, de igual manera la unica forma de que una tarea se coloque en la lista de finalizados o listos sería que comprobemos que funciona correctamente y cumple con su función.
