1. Escenario elegido:
La opción elegida por el equipo es la C, debido a que nos resultó interesante y un escenario a el cual no nos sometimos antes, nos resulta un desafío que nos puede hacer aprender mucho más, debido a que con anterioridad ya hemos realizado turneros o sistemas de gestion de inventario. Nuestra intención en este proyecto es utilizar nuevas herramientas y aprender a usar muchas de las cuales no tuvimos tiempo con anterioridad, como el caso de github, y aprender a utilizar otros lenguajes distintos a python, en este caso intentaremos usar java script.

2. Metodología elegida: 
La metodología de desarrollo elegida es scrum, debido a que consideramos que para este proyecto se ajusta mucho mas a nuestros horarios, el hecho de realizar reuniones cada determinado tiempo nos va a ayudar a ajustar nuestra forma de trabajo y a ser mas eficientes con las tareas que le corresponden a cada uno.
También nos ayudará al hecho de adaptarnos a un lenguaje nuevo el cual no tenemos familiarizado, y solucionar los errores y entender finalmente el lenguaje.

5. Acuerdos de trabajo: 
Nuestros principales acuerdos como equipo de trabajo son: comunicación via Whatsapp debido a que es la forma mas accesibe para la comunicación rapida, Reuniones cada 2 días realizadas en Discord con objetivo de direccionar y eficientizar el trabajo de cada uno, la frecuencia de comitts será cada vez que comprobemos que el módulo a agregar cumple con las funciones requeridas. El criterio para mover las tarjetas será estrictamente cuando la tarea esté en curso, de igual manera la unica forma de que una tarea se coloque en la lista de finalizados o listos sería que comprobemos que funciona correctamente y cumple con su función.
